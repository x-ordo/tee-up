# Feature Specification: UX 접근성 및 디자인 일관성 개선

**Feature Branch**: `001-ux-a11y-fixes`
**Created**: 2025-12-02
**Status**: Draft
**Input**: User description: "UX 접근성 및 디자인 일관성 개선 - P0/P1 이슈 해결"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 모바일 사용자 콘텐츠 접근성 (Priority: P1)

모바일 기기로 프로 프로필 페이지를 방문한 골퍼가 화면 하단의 플로팅 CTA 버튼에 의해 콘텐츠가 가려지지 않고 모든 정보를 확인할 수 있어야 합니다.

**Why this priority**: 모바일 사용자가 전체 사용자의 70% 이상을 차지하며, 현재 플로팅 버튼이 화면의 20%를 가려 핵심 정보 접근을 방해합니다. 이는 예약 전환율에 직접적인 영향을 미칩니다.

**Independent Test**: 모바일 기기(375px 너비)에서 프로 프로필 페이지의 모든 섹션을 스크롤하여 콘텐츠가 완전히 보이는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 모바일 기기에서 프로 프로필 페이지 접속, **When** 페이지 하단까지 스크롤, **Then** 모든 콘텐츠가 CTA 버튼에 가려지지 않고 표시됨
2. **Given** 모바일 기기에서 프로 프로필 페이지 접속, **When** CTA 버튼 영역 확인, **Then** 버튼이 콘텐츠와 겹치지 않거나 최소화/닫기 옵션 제공
3. **Given** 320px 너비의 작은 화면, **When** 프로 프로필 페이지 접속, **Then** CTA가 단일 버튼으로 축소되어 화면 점유율 최소화

---

### User Story 2 - 키보드 사용자 전체 기능 접근 (Priority: P1)

키보드만 사용하는 사용자(시각 장애인 포함)가 모든 인터랙티브 요소를 탐색하고 활성화할 수 있어야 합니다.

**Why this priority**: WCAG AA 준수는 법적 요구사항이며, 키보드 접근성 부재는 장애인 사용자의 완전한 서비스 이용을 차단합니다.

**Independent Test**: Tab 키만으로 홈페이지, 프로필 페이지, 예약 모달의 모든 인터랙티브 요소를 탐색하고 Enter로 활성화합니다.

**Acceptance Scenarios**:

1. **Given** 홈페이지에서 키보드 사용, **When** Tab 키로 탐색, **Then** 모든 링크와 버튼이 시각적으로 구분되는 포커스 표시 제공
2. **Given** 예약 모달이 열린 상태, **When** Tab 키로 탐색, **Then** 포커스가 모달 내부에만 순환하고 배경 요소로 이동하지 않음
3. **Given** 모달이 열린 상태에서 Escape 키 입력, **When** 키 입력 완료, **Then** 모달이 닫히고 포커스가 모달을 연 버튼으로 복귀

---

### User Story 3 - 디자인 시스템 일관성 유지 (Priority: P2)

개발자와 디자이너가 플랫폼 전체에서 일관된 시각적 경험을 제공하기 위해 모든 UI 요소가 정의된 디자인 토큰을 사용해야 합니다.

**Why this priority**: 하드코딩된 색상값은 디자인 시스템 변경 시 일괄 수정이 불가능하고, 브랜드 일관성을 해칩니다.

**Independent Test**: 에러 페이지와 카카오톡 버튼에서 하드코딩된 색상이 제거되고 디자인 토큰으로 대체되었는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 시스템 에러 발생, **When** 에러 페이지 표시, **Then** 에러 페이지의 모든 색상이 디자인 시스템 변수 사용
2. **Given** 프로 프로필 페이지 접속, **When** 카카오톡 버튼 확인, **Then** 카카오 브랜드 색상이 별도 변수로 정의되어 디자인 시스템과 분리 관리됨
3. **Given** 디자인 토큰(accent color) 변경, **When** 페이지 새로고침, **Then** 모든 accent 색상 요소가 일괄 변경됨

---

### User Story 4 - 스크린 리더 사용자 정보 접근 (Priority: P2)

스크린 리더를 사용하는 시각 장애인이 모든 인터랙티브 요소의 목적을 이해하고 동적 콘텐츠 변경을 인지할 수 있어야 합니다.

**Why this priority**: WCAG AA 준수 요구사항이며, 아이콘 전용 버튼과 동적 알림에 적절한 접근성 레이블이 필요합니다.

**Independent Test**: 스크린 리더로 홈페이지와 프로필 페이지의 모든 버튼, 링크, 알림을 탐색하여 적절한 설명이 제공되는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 스크린 리더로 아이콘 전용 버튼 탐색, **When** 버튼에 포커스, **Then** 버튼의 기능을 설명하는 레이블이 읽힘
2. **Given** 폼 제출 후 에러 발생, **When** 에러 메시지 표시, **Then** 스크린 리더가 즉시 에러 내용을 자동으로 읽음
3. **Given** 로딩 상태 시작, **When** 로딩 스피너 표시, **Then** 스크린 리더가 "로딩 중" 상태를 알림

---

### User Story 5 - 관리자 테이블 모바일 접근성 (Priority: P3)

모바일 기기로 관리자 페이지에 접속한 관리자가 데이터 테이블을 스크롤하여 모든 컬럼의 정보를 확인할 수 있어야 합니다.

**Why this priority**: 관리자도 모바일에서 긴급 상황 대응이 필요하며, 현재 테이블이 오버플로우되어 사용 불가합니다.

**Independent Test**: 모바일 기기에서 관리자 채팅 페이지의 테이블을 좌우 스크롤하여 모든 컬럼을 확인합니다.

**Acceptance Scenarios**:

1. **Given** 모바일 기기에서 관리자 채팅 페이지 접속, **When** 테이블 영역 확인, **Then** 수평 스크롤 표시기가 보이고 터치로 스크롤 가능
2. **Given** 테이블 스크롤 중, **When** 첫 번째 컬럼(ID/이름) 확인, **Then** 식별 컬럼이 고정되어 항상 보임 (선택적)

---

### Edge Cases

- 320px 미만의 극소형 화면에서 CTA 버튼이 어떻게 표시되는가? → 단일 아이콘 버튼으로 최소화
- 자바스크립트가 비활성화된 환경에서 포커스 트랩이 동작하는가? → 기본 브라우저 동작에 의존
- 고대비 모드에서 포커스 링이 충분히 보이는가? → 시스템 고대비 설정 존중
- RTL(우에서 좌) 언어 환경에서 레이아웃이 유지되는가? → 현재 범위 외 (한국어/영어만 지원)

## Requirements *(mandatory)*

### Functional Requirements

**모바일 UX (P0)**
- **FR-001**: 프로필 페이지의 플로팅 CTA는 화면 하단 콘텐츠를 10% 이상 가리면 안 됨
- **FR-002**: 320px-768px 화면에서 CTA는 단일 버튼으로 축소되어야 함
- **FR-003**: 플로팅 CTA는 닫기 또는 최소화 옵션을 제공해야 함

**키보드 접근성 (P0/P1)**
- **FR-004**: 모든 인터랙티브 요소는 Tab 키로 접근 가능해야 함
- **FR-005**: 모든 인터랙티브 요소는 포커스 시 시각적으로 구분되는 표시를 제공해야 함
- **FR-006**: 모달은 포커스 트랩을 구현하여 키보드 포커스가 모달 내부에서만 순환해야 함
- **FR-007**: 모달은 Escape 키로 닫을 수 있어야 함

**디자인 시스템 일관성 (P0)**
- **FR-008**: 에러 페이지의 모든 색상은 디자인 시스템 변수를 사용해야 함
- **FR-009**: 카카오톡 브랜드 색상은 별도 변수로 정의하여 관리해야 함
- **FR-010**: 버튼 크기와 패딩은 정의된 크기 변형(small, medium, large)을 사용해야 함

**스크린 리더 접근성 (P1)**
- **FR-011**: 아이콘 전용 버튼은 접근성 레이블을 제공해야 함
- **FR-012**: 동적 콘텐츠 변경(에러, 성공, 로딩)은 스크린 리더에 자동 알림되어야 함
- **FR-013**: 장식용 이미지/아이콘은 스크린 리더에서 무시되어야 함

**관리자 테이블 (P1)**
- **FR-014**: 관리자 테이블은 모바일에서 수평 스크롤이 가능해야 함
- **FR-015**: 스크롤 가능한 테이블은 스크롤 가능 여부를 시각적으로 표시해야 함

### Key Entities

- **디자인 토큰**: 색상, 간격, 타이포그래피 등 재사용 가능한 디자인 값
- **인터랙티브 요소**: 버튼, 링크, 폼 컨트롤 등 사용자 입력을 받는 UI 요소
- **포커스 상태**: 키보드 탐색 시 현재 선택된 요소를 나타내는 시각적 표시

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 모바일(375px) 프로필 페이지에서 모든 콘텐츠의 100%가 CTA에 가려지지 않고 접근 가능
- **SC-002**: 키보드만으로 홈페이지 → 프로 디렉토리 → 프로 프로필 → 예약 모달 완료까지 전체 흐름 수행 가능
- **SC-003**: 자동화된 접근성 검사 도구에서 0개의 치명적(critical) 오류
- **SC-004**: 모든 페이지에서 하드코딩된 색상값 0개 (디자인 토큰 100% 사용)
- **SC-005**: 스크린 리더 사용자가 모든 버튼의 목적을 3초 이내에 파악 가능
- **SC-006**: 관리자 모바일 페이지에서 테이블의 모든 컬럼 접근 가능 (스크롤 포함)
- **SC-007**: WCAG AA 색상 대비 기준(4.5:1 for text, 3:1 for large text) 100% 충족

## Assumptions

- WCAG 2.1 AA 수준을 목표로 함 (AAA는 현재 범위 외)
- 지원 브라우저: Chrome, Safari, Firefox, Edge 최신 2개 버전
- 지원 스크린 리더: VoiceOver (macOS/iOS), NVDA (Windows)
- 카카오톡 버튼의 브랜드 가이드라인(노란색 #FEE500)은 유지하되, 디자인 시스템과 분리 관리
- 관리자 페이지는 주로 데스크톱에서 사용되나, 모바일 기본 접근성은 보장
