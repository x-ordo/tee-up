# Screen: Error Pages (에러 페이지)

**Route**: `/404`, `/500`, `/error`
**Status**: Implemented
**Last Updated**: 2025-12-30
**Priority**: P1
**Dependencies**: None

---

## 1. 화면 개요

### 목적
사용자에게 에러 상황을 안내하고, 적절한 다음 행동을 제시한다.

### 에러 유형

| 코드 | 이름 | 설명 |
|------|------|------|
| 404 | Not Found | 페이지를 찾을 수 없음 |
| 500 | Server Error | 서버 오류 |
| 403 | Forbidden | 접근 권한 없음 |
| 401 | Unauthorized | 인증 필요 |

## 3. 주요 컴포넌트

### 3.1 ErrorIcon

**에러 아이콘**

| Error | Icon | Color |
|-------|------|-------|
| 404 | 🔍 (돋보기) | `--tee-ink-light` |
| 500 | ⚠️ (경고) | `--tee-warning` |
| 403 | 🔒 (자물쇠) | `--tee-ink-light` |
| 401 | 🔑 (열쇠) | `--tee-ink-light` |

### 3.2 ErrorMessage

**에러 메시지**

| Element | Description |
|---------|-------------|
| 제목 | 주요 에러 메시지 (h1) |
| 설명 | 상세 설명 텍스트 |
| 스타일 | 중앙 정렬, `--tee-ink-strong` |

### 3.3 ErrorActions

**액션 버튼**

| Button | Style | Action |
|--------|-------|--------|
| 다시 시도 | Primary | `window.location.reload()` |
| 홈으로 돌아가기 | Secondary | `/` 이동 |
| 로그인하기 | Primary | `/auth/login` 이동 |

### 3.4 HelpSection

**도움말 섹션**

| Element | Description |
|---------|-------------|
| 구분선 | `--tee-stone` 색상 |
| 제안 | 불릿 리스트로 다음 행동 안내 |
| 연락처 | 고객센터 이메일 |

## 5. 인터랙션 정의

### 5.1 다시 시도

1. **Given** 500 에러 페이지가 표시될 때
2. **When** "다시 시도" 버튼을 클릭하면
3. **Then** 현재 페이지가 새로고침된다

### 5.2 자동 리포팅

1. **Given** 500 에러가 발생할 때
2. **Then** 에러 정보가 로깅 서비스로 전송된다

- 중앙 정렬
- 최대 너비 480px
