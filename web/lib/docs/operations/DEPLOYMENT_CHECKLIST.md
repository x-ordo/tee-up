# DEPLOYMENT_CHECKLIST.md

## 배포 체크리스트

> **본 문서는 프로덕션 배포 전 확인해야 할 사항들을 정의합니다.**

---

## 1. 배포 전 체크리스트

### 1.1 코드 검증

- [ ] 모든 PR이 승인되고 머지됨
- [ ] 모든 테스트가 통과함 (Unit, Integration, E2E)
- [ ] 린트 에러 없음
- [ ] 타입 체크 통과
- [ ] 보안 스캔 통과 (npm audit)

### 1.2 빌드 검증

- [ ] 프론트엔드 빌드 성공
- [ ] 백엔드 빌드 성공
- [ ] 빌드 사이즈 적정 (Frontend < 500KB gzip)
- [ ] 환경 변수 설정 확인

### 1.3 데이터베이스

- [ ] 마이그레이션 스크립트 준비됨
- [ ] 마이그레이션 롤백 스크립트 준비됨
- [ ] 백업 완료
- [ ] RLS 정책 확인

### 1.4 문서

- [ ] CHANGELOG.md 업데이트
- [ ] API 문서 업데이트 (변경사항이 있는 경우)
- [ ] 버전 번호 업데이트

---

## 2. 배포 중 체크리스트

### 2.1 스테이징 배포

- [ ] 스테이징 환경에 배포
- [ ] 스모크 테스트 통과
- [ ] 기능 테스트 통과
- [ ] 성능 테스트 통과

### 2.2 프로덕션 배포

- [ ] 배포 승인 획득
- [ ] 프론트엔드 배포
- [ ] 백엔드 배포
- [ ] 데이터베이스 마이그레이션 (필요시)

---

## 3. 배포 후 체크리스트

### 3.1 헬스 체크

- [ ] 프론트엔드 접속 확인 (https://teeup.kr)
- [ ] 백엔드 헬스 체크 (https://api.teeup.kr/health)
- [ ] 데이터베이스 연결 확인
- [ ] 외부 서비스 연결 확인 (Supabase, Cloudinary)

### 3.2 기능 확인

- [ ] 홈페이지 로딩
- [ ] 프로 목록 조회
- [ ] 프로 상세 페이지
- [ ] 로그인/회원가입 (Phase 2)
- [ ] 채팅 기능 (Phase 2)

### 3.3 모니터링

- [ ] Sentry 에러 모니터링 활성화
- [ ] 성능 지표 정상 (LCP < 2.5s)
- [ ] 에러율 정상 (< 1%)
- [ ] 알림 설정 확인

### 3.4 커뮤니케이션

- [ ] 팀 Slack 공지
- [ ] 릴리스 노트 발행
- [ ] 고객 공지 (주요 변경사항의 경우)

---

## 4. 롤백 절차

### 4.1 롤백 조건

- 에러율 > 5%
- 주요 기능 장애
- 데이터 손실 위험

### 4.2 롤백 단계

```bash
# 1. 프론트엔드 롤백
vercel rollback

# 2. 백엔드 롤백
railway rollback

# 3. 데이터베이스 롤백 (필요시)
supabase db reset --version <previous-version>

# 4. 롤백 완료 후 알림
```

### 4.3 롤백 후

- [ ] 원인 분석
- [ ] 핫픽스 준비
- [ ] 포스트모템 작성
